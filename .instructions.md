# Canonical Instructions

This file is the canonical instruction source for this repository.

## Load Order

1. Read rules from `.agent-context/rules/`.
2. Read stack profile from `.agent-context/stacks/` (prefer `.agent-context/stacks/php.md` for this repo).
3. Read blueprints from `.agent-context/blueprints/` only for scaffolding tasks.
4. Read domain skills from `.agent-context/skills/`.
5. Read request templates from `.agent-context/prompts/` when applicable.
6. Apply defaults from `.agent-context/profiles/`, state context from `.agent-context/state/`, and thresholds from `.agent-context/policies/`.

## Skill Routing (Auto Detect)

When a request matches these keywords, load the matching skill wrapper in `.github/skills/`:

- Backend API, Laravel logic, validation, migration, query, auth, security hardening: `.github/skills/agentic-backend/SKILL.md`
- Frontend UI, Blade, CSS, responsive, accessibility, conversion: `.github/skills/agentic-frontend/SKILL.md`
- End-to-end feature touching backend and frontend: `.github/skills/agentic-fullstack/SKILL.md`
- CLI workflow, project init, upgrade flow, command safety: `.github/skills/agentic-cli/SKILL.md`
- Packaging, publish, rollback, compatibility release checks: `.github/skills/agentic-distribution/SKILL.md`
- Code review, security audit, benchmark, release readiness: `.github/skills/agentic-review-quality/SKILL.md`

## Delivery Defaults

- Refactor and fix requests: share a brief plan, then execute safely.
- Avoid destructive git operations unless explicitly approved.
- Prefer minimal, test-backed changes.
- Run `.agent-context/review-checklists/pr-checklist.md` before completion for non-trivial changes.
